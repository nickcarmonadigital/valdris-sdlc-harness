## What changed materially

The latest harness adds/strengthens the **Prompt Library** and the master-flow enforcement pattern. Compared to the earlier pass, this makes the harness more universalizable because the core logic is now explicit:

```text
lane -> stage -> gate -> artifact -> finish-line validator -> answer contract
```

## Integration design

1. **Commissioning interview** collects the human/team context Graphify cannot infer.
2. **Graphify scan** fills code topology, symbols, entrypoints, blast radius, and graph freshness.
3. **Harness generator** instantiates the universal core into a project adapter.
4. **Coherence check** ensures docs/stages/gates are internally consistent.
5. **Prompt Library generator** creates project-specific master-flow prompts and anti-skip nudges.
6. **Kanban bootstrapping** creates lanes/cards for ongoing work.
7. **Finish-line validator** blocks “done” unless required gate artifacts exist.

## Product insight

The Utari harness is not just docs. It is a filesystem-native **software delivery control system for agents**. Universalization means turning its structure into a generator and validator.
